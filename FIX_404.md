# 🔧 Hướng dẫn sửa lỗi 404 cho forgot-password

## ⚠️ Nguyên nhân
Lỗi 404 xảy ra vì server backend chưa được **restart** sau khi thêm route mới.

## ✅ Giải pháp

### Bước 1: Dừng server backend hiện tại
1. Mở terminal đang chạy server backend
2. Nhấn `Ctrl + C` để dừng server

### Bước 2: Khởi động lại server
```bash
cd backend
node server.js
```

### Bước 3: Kiểm tra log
Sau khi start server, bạn sẽ thấy log như sau:
```
=== Server running on port 5000 ===
Connected to MySQL database
✓ Auth routes loaded successfully
  - POST /api/auth/login
  - POST /api/auth/register
  - POST /api/auth/forgot-password
Test endpoint: http://localhost:5000/api/test
Auth endpoint: http://localhost:5000/api/auth/forgot-password
```

**Nếu thấy log này** → Route đã được load thành công ✅

**Nếu không thấy log này hoặc có lỗi** → Kiểm tra:
- File `.env` có đúng cấu hình không
- MySQL có đang chạy không
- Database `chamcong` đã được tạo chưa

### Bước 4: Test API
Sau khi server chạy, test bằng cách:

1. **Test server** (GET):
   - Mở browser: `http://localhost:5000/api/test`
   - Kỳ vọng: `{"message": "Server is running"}`

2. **Test forgot-password** (POST):
   - Dùng Postman hoặc curl:
   ```bash
   curl -X POST http://localhost:5000/api/auth/forgot-password \
     -H "Content-Type: application/json" \
     -d '{"ma_nhan_vien":"1","default_password":"123@123a","new_password":"newpass123"}'
   ```

### Bước 5: Kiểm tra database
Đảm bảo database đã được tạo với cấu trúc đúng:

```bash
# Mở MySQL và chạy:
mysql -u root -p < backend/database.sql
```

Hoặc trong MySQL Workbench, mở và chạy file `backend/database.sql`

## 📝 Kiểm tra nhanh
- [ ] Server backend đã được restart
- [ ] Log hiển thị route `/api/auth/forgot-password`
- [ ] Test endpoint `/api/test` hoạt động
- [ ] Database `chamcong` đã được tạo
- [ ] Bảng `nhanvien` có dữ liệu mẫu

## 🐛 Nếu vẫn còn lỗi

1. **Kiểm tra console backend** khi gọi API:
   - Có log `Forgot password endpoint called` không?
   - Có log `404 - Route not found` không?

2. **Kiểm tra browser console**:
   - Xem network tab
   - Kiểm tra request URL có đúng không
   - Kiểm tra response status code

3. **Kiểm tra file `.env`**:
   ```
   DB_HOST=localhost
   DB_USER=root
   DB_PASSWORD=your_password
   DB_DATABASE=chamcong
   PORT=5000
   JWT_SECRET=your_secret_key
   ```
